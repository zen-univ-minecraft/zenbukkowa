# Commit Policy

## Frequency

- Commit after every coherent batch that passes verification.
- Do not accumulate unverified changes.

## Messages

- Use imperative mood.
- Prefix with affected area: `docs:`, `feat:`, `fix:`, `test:`.
- Example: `feat: add AREA_DEPTH skill and TERRA gating`

## Atomicity

- One batch = one commit.
- A batch contains docs changes + code changes + tests for a single feature.
