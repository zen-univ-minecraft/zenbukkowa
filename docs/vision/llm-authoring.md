# LLM Authoring Rules

## Formatting Rules

- Use stable headings (`Goal`, `Rules`, `Contract`, `Verification`).
- Keep one requirement per bullet where possible.
- Use relative links only.
- Keep canonical definitions in one owner file.

## Topology Rules

- Every docs directory has one `README.md` table of contents.
- Every docs directory has multiple children.
- Parent `README.md` files must be updated in the same batch as child edits.

## Length Rules

- Docs files: `<= 300` lines.
- Authored source files: `<= 200` lines.

## Terminology Rules

- Use `points` for currency generated from block breaking.
- Use `category` for one of the six point types: `TERRA`, `MINERAL`, `ORGANIC`, `AQUATIC`, `VOID`, `CROP`.
- Use `skill` for a purchasable player upgrade.
- Use `tier` for a skill level.
- Use `area` for the break radius and depth.
- Use `structure bonus` for one-time rewards from structure destruction.
