# Product

## Goal

Define player-facing behavior for point categories, skills, block breaking, menus, scoreboard, event flow, effects, and localization.

## Rules

1. Every player-visible behavior change updates this section first.
2. Menu flows and break outcomes must describe the same results.
3. Success/failure messaging expectations must remain explicit.
4. Destructive actions require explicit dedicated flows.
5. Skill purchases are final; no refunds.
